KEY OBSERVATIONS
================================================================================

1. Gas is virtually IDENTICAL to cold settle (170,266 vs 170,263)
2. Only 3 gas difference — within measurement noise
3. All borrow-position storage is already warm in BOTH cases
4. No health check, no rate limiting, no PoW check
5. The "cold" vs "warm" distinction is meaningless for settle
6. This confirms that settle has no user-specific cold storage overhead

================================================================================
WHY COLD == WARM FOR SETTLE
================================================================================

The "cold" settle user (cold2) already has warm borrow-position storage
because they borrowed in setUp. The "warm" settle user (self) also has
warm storage from their prior borrow.

Since settle only BURNS borrow position tokens (never writes to new
user-specific slots), there are no cold SSTOREs in either case.

The only path to a truly "cold" settle would be if a user could settle
without ever having borrowed — which is impossible (you must borrow
before you can settle).
