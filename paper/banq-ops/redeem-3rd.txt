================================================================================
WHY COLD == WARM FOR REDEEM
================================================================================

The "cold" redeem user (cold2) already has warm supply-position storage
because they supplied in setUp. The "warm" redeem user (self) also has
warm storage from their prior supply.

Since redeem only BURNS supply position tokens (never writes to new
user-specific slots), there are no cold SSTOREs in either case.

The only path to a truly "cold" redeem would be if a user could redeem
without ever having supplied — which is impossible (you must supply
before you can redeem).
